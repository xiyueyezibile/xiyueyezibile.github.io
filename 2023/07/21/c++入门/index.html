第一个c++程序

using namespace std是命名空间

命名空间

为什么要有命名空间？

我们来看看c语言

这样程序是可以正常执行的，但是我们加上一个头文件

程序就会报错，因为rand这个变量和头文件stdlib.h里面的变量命名冲突了

所以在c语言中，是存在命名冲突的问题的

c++命名空间就是解决命名冲突这个问题的

语法

默认是在全局查找，不会进入命名空间查找

using namespace acm会把命名空间全部展开

展开之后就可以进入里面查找，但这个操作是危险的

我们可以只展开一部分

using acm::Add只展开了Add函数

注：同名的命名空间会进行合并，不用担心命名空间名冲突

std

std是c++标准库的命名空间

头文件iostream里面的cout,endl等都封装在std这个命名空间里面

基础语法

cout,endl

cout 控制台打印，会自动识别类型

endl 可以相当于\n

cin 输入

cin可以自动识别类型

<< 流插入运算符

>>流提取

缺省参数

int a = 1就是缺省参数

如果你不传参数，a就会等于缺省值，这里是1

如果你传了参数，a就会等于传的值

缺省又分为全缺省和半缺省

全缺省就是全部都是缺省参数

半缺省有缺省参数也有普通参数，半缺省必须从右往左给缺省值

注：声明和定义不能同时给缺省参数

函数重载

概念

c++允许在同一作用域下声明多个相同名字的函数，但这些函数的形参列表不同（个数，类型或顺序不同）

为什么c++支持函数重载而c语言不支持？

过程

test.cpp

预处理：头文件展开/宏替换/去掉注释/条件编译

test.i

编译：检查语法，生成汇编代码

test.s

汇编：将汇编代码生产二进制机器码

test.o

链接：合并链接，生成可执行程序

c语言是直接用函数名字充当符号表里面的名字

c++则有一套函数名修饰规则,会把类型带进去

所以函数重载类型不同，个数不同，位置不同，c++就可以区分

函数重载和缺省参数一起会导致一个问题

c++关键字

c++总计63个关键字，c语言32个关键字

引用

引用不会新定义一个变量，而是给已存在的变量取一个别名，编译器不会为引用变量开辟内存空间，它和它引用的变量公用一片内存空间。

语法

特性

  1. 引用在定义时必须初始化
  2. 一个变量可以有多个引用
  3. 引用一旦引用一个实体，不能再引用其他实体（不能再变）

权限

  • 权限可以缩小
  • 权限可以平移
  • 权限不能放大
