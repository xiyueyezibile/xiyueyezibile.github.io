简介

React Router 是一个基于 React 之上的强大路由库，它可以让你向应用中快速地添加视图和数据流，同时保持页面与 URL 间的同步。

工作方式

是在组件树顶层放一个 Router 组件，然后在组件树中散落着很多 Route 组件（注意比 Router 少一个“r”），顶层的 Router 组件负责分析监听 URL 的变化，在它保护伞之下的 Route 组件可以直接读取这些信息。Router 是“提供者”，Route是“消费者”，常用的有BrowserRouter和HashRouter

这就涉及到选择用哪种Router的问题，以往都是有人给我们搭好项目框架，可以直接使用，也不关心路由，当你一个人搭建框架的时候，你就会考虑这个问题了？为什么要用这个路由？？

BrowserRouter

在 React Router 中，最外层的 API 通常就是用 BrowserRouter。BrowserRouter 的内部实现是用了 history 这个库和 React Context 来实现的，所以当你的用户前进后退时，history 这个库会记住用户的历史记录，这样需要跳转时可以直接操作。

1.基于history模式：页面跳转原理是使用了HTML5为浏览器全局的history对象新增了两个API，包括 history.pushState、history.replaceState
2.更加优雅： 直接拼接路径；如：www.abc.com/xx
3.后端需做请求处理： 切换路由后，请求接口路径会发生变化，后端需要配合，做处理

HashRouter

这一种看起来不自然，但是实现更简单。只有一个路径 /，通过 URL 后面的 # 部分来决定路由，/#/ 对应 Home 页，/#/about 对应 About 页。因为URL中#之后的部分是不会发送给服务器的，所以，无论哪个 URL，最后都是访问服务器的 / 路径，服务器也只需要返回同样一份 HTML就可以，然后由浏览器端解析 # 后的部分，完成浏览器端渲染。HashRouter支持这种URL。
1.基于hash模式：页面跳转原理是使用了location.hash、location.replace；和vue router的hash模式实现一致
2.比较丑：在域名后，先拼接/#，再拼接路径；也就是利用锚点，实现路由的跳转；如：www.abc.com/#/xx

使用场景

HashRouter

项目部署在内网：如To B的项目、本公司业务人员用的项目等等

BrowserRouter

项目部署在公网：如To C的项目、面向大众的项目，url路径美观点当然更好，但后端需要做处理

API

path

当URL路径与path匹配时对应组件将被挂载

默认采用模糊匹配，也就是只要包含该路径则挂载

如：path=”/home/about”

则/home下的组件<Home>也将被挂载和他的子路由about也会被挂载

当前，前提是他们不能是同级路由，同级路由同一时间只会挂载一个

动态区段

如果路径段以:开头，则会成为动态段

如：path=”/home/:id”

/home/123或者/home/234都会匹配该路径，后面一段路径是动态的

但/home则不会匹配

你也可以在路由路径中包含多个动态路段

如：path=”/home/:id/c/:a”

可选段

在路径端末尾添加?则表示可选段

如:path=”/home/id?”

或者使用可选的动态段path=”/home/:id?”

星形段

以*结尾的路径段，他将匹配后面任何字符,一般使用*来配置404页面

布局路由

省略path 会使得此路由成为布局路由，他参加UI嵌套，但不提供任何路径

index

默认子路由, 默认在父路由路径中渲染

element

路径对应时采用渲染的React组件

路由跳转

Link

通过使用 JSX 元素，如 <Link />、<NavLink /> 等，设置 to 属性，点击该元素来进行跳转的方式

在页面表现效果为一个a链接标签,但和a标签不一样

使用锚点元素实现，在每次点击时，页面被重新加载，React Router提供了<Link>组件用来避免这种状况发生。当 你点击<Link>时，url会更新，组件会被重新渲染，但是页面不会重新加载

API

to：要跳转到的路径，可以通过 ../ 的方式跳转到上层组件/路径

relative：默认情况下，链接相对于router层次结构，也就是说当你通过../的方式跳转是跳转上层路由，

​ 但有时候你希望是跳转上层路径，也就是基于path层次结构，需要给relative设置为’path’

preventScrollReset: 跳转时窗口是否将滚动位置重置为窗口

replace：是否直接替换历史堆栈中的当前路由

state：为历史记录中的新位置设置状态值，然后可以通过useLocation()访问

NavLink

是一种特殊的 <Link />，用来判断当前路由是“活跃的”还是“待定的”。

活跃状态则是路径匹配的时候

默认情况下，当组件处于活动状态时，会将类添加到组件中，以便您可以使用 CSS 设置其样式

className:

类似于普通的类名，但您也可以向它传递一个函数，以根据链接的活动和挂起状态自定义应用的类名。

不光是className，style和内部也可以传递该函数

end:

将active 和 pending 状态的匹配逻辑更改为仅与 NavLink 路径的“结束”匹配。

如果原本路径是/home，则/home/1也会视为活跃

但有了end之后则不会，只有路径完全匹配才会视为

useNavigate()

useNavigate 钩子返回一个函数，让您可以以编程方式导航

navigate 函数有两个签名：

  • 要么传递一个 To 值（与 <Link to> 相同的类型），带有可选的第二个 { replace, state } 参数，要么
  • 传递您想要在历史堆栈中前进的增量。例如， navigate(-1) 等同于点击后退按钮。
