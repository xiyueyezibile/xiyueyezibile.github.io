npm包管理器

发表于2023-05-27|更新于2023-05-29|nodeJS
|阅读量:

package.json是包的描述文件

node中通过该文件对项目进行描述

每一个node项目必须有package.json

package.json

name(必备)：包的名称，可以包含小写字母，_和-

version(必备)：包的版本，需遵循x.x.x格式

description：对包的介绍

main:主文件

author：作者

license：版权的协议

scripts(脚本)：可以自定义一些命令，然后可以通过npm进行执行

规则：

    • 版本从1.0.0开始
    • 修复错误，兼容旧版（补丁）1.0.1、1.0.2
    • 添加功能，兼容旧版（小更新）1.1.0
    • 更新功能，影响兼容（大更新）2.0.0

在终端通过npm init进行初始化package.json（需要回答问题）

npm init -y表示所有的都按默认的来

npm install(i) 包名 将指定包下载到当前项目中

如果我们下载lodash，下载完成后会弹出

添加了1个包，移除了8个包，检查了10个包，发现了0个错误

install时发生了什么？

1.将包下载到当前项目的node_modules下

2.会在package.json中的dependencies（依赖）添加一个新的属性

如果此时删除了包，npm i就会自动下载所有依赖

3.会自动添加package-lock.json文件

引入从npm下载的包时，直接写包名就可以

我们安装时也可以指定版本：

安装旧版本

安装大于该版本的版本

npm install 包名 -g 全局安装

全局安装时安装到计算机中

通常装的都是一些工具

所有的组件可以通过npm uninstall xxx来完成卸载。

npm镜像

npm的仓库的服务器位于国外，有时并不是那么好使

为了解决这个问题，可以在npm中配置一个镜像服务器

镜像的配置：

1.在系统中安装cnpm

然后把所有的npm改成cnpm就行，你想用原版就npm，镜像就cnpm

2.彻底修改npm仓库地址（推荐）
