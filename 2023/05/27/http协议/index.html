http协议

发表于2023-05-27|更新于2023-05-27|nodeJS
|阅读量:

（一）当在浏览器中输入地址以后发生了什么？

1.DNS解析，获取网站的ip地址

2.浏览器需要和服务器建立连接(tcp/ip)

3.向服务器发送请求（http协议）

4.服务器处理请求，并返回响应

5.浏览器将响应的页面渲染

6.断开和服务器的链接

https:// lilichao.com / hello/index.html

https:// 协议名

lilichao.com 域名（domain）

每个服务器都有自己的唯一标识（ip地址），但是不好记忆，所以有域名

/ hello/index.html 网站资源路径

（二）客户端如何和服务器建立（断开）链接

​ -通过三次握手和四次挥手。

​ -三次握手

1.客户端发送连接请求（SYN）。

2.服务器收到连接请求，向客户端返回信息(SYN,ACK同意)。

3.客户端向服务器发送同意连接的信息（ACK）。

​ -四次挥手

1.客户端向服务器发送请求，服务器数据发送完毕，请求断开连接（FIN）。

2.服务器向客户端返回数据，知道了(ACK)。

3.服务器向客户端返回数据，收完了，可以断开连接(FIN,ACK)。

4.客户端向服务器发数据，知道了，可以断开了(ACK)。

TCP/IP协议族

TCP/IP这组协议规定了互联网中所有的通信的细节

​ -网络通信的过程

收数据时

应用层：软件的层面，浏览器 服务器都属于应用层

传输层：负责对数据进行拆分，把大数据分为一个一个小包

网络层：负责给数据包添加信息

数据链路层：发送信息

发数据时相反

​ -HTTP协议就是应用层协议，用来规定客户端和服务器间通信的报文格式

报文？

浏览器和服务器之间的通信基于请求和响应

​ -浏览器向服务器发送请求（request）

​ -服务器向浏览器返回响应（response）

请求和响应就是报文

而HTTP协议就是对报文格式进行规定。

-请求报文（request）

​ 格式：

​ 1.请求首行

GET /index.html?username=sunwukong HTTP/1.1

通过form里的method属性更改请求，默认get

get请求，post请求

get请求用来向服务器请求资源

post请求主要用来向服务器发送数据

/index.html?username=sunwukong表示请求资源的路径

？后面的内容叫查询字符串，查询字符串是一个名值对结构

多个查询字符串之间用&符分隔

get请求通过查询字符串将数据发送给服务器

​ 但查询字符串会在浏览器地址栏中直接显示，所以安全性较差

​ 同时url地址长度有限制，所以get请求无法发送较大的数据

post通过请求体发送数据

​ 安全性较好，请求体大小没有限制

HTTP/1.1 协议版本

​ 2.请求头

也是名值对结构，告诉服务器我们浏览器的信息

Accept 浏览器可以接受的文件类型

Accept-Encoding 浏览器允许的压缩的编码

Accept-Language 浏览器可接收的语言 zh-CN（简体或繁体中文）

User-Agent 用户代理，它是一段用来描述浏览器信息的字符串

​ 3.空行

用来分隔请求头和请求体

​ 4.请求体（get请求没有）

post请求通过请求体来发送数据

响应报文：

1.响应首行

HTTP/1.1 304 Not Modified

304 响应状态码

Not Modified 对响应状态码的描述

响应状态码规则：

1xx 请求处理中

2xx 表示成功

3xx 表示请求的重定向

4xx 表示客户端错误

5xx 表示服务器错误

2.响应头

Content-Type 描述响应体的类型

Content-Length 描述响应大小

3.空行

分隔响应头和响应体

4.响应体

​ 响应体是服务器返回给客户端的内容
